            for infra_req in understanding.get("infrastructure_requests", []):

Change the `_run_terraform` function to pass `env=os.environ.copy()`:
```python
async def _run_terraform(cmd: list[str], cwd: str) -> tuple[int, str, str]:
    """Run a terraform command, return (returncode, stdout, stderr)."""
    proc = await asyncio.create_subprocess_exec(
        *cmd,
        cwd=cwd,
        stdout=asyncio.subprocess.PIPE,
        stderr=asyncio.subprocess.PIPE,
        env=os.environ.copy(),
    )
    stdout_bytes, stderr_bytes = await proc.communicate()
    return (
        proc.returncode,
        stdout_bytes.decode("utf-8", errors="replace"),
        stderr_bytes.decode("utf-8", errors="replace"),
    )
```

Actually, `asyncio.create_subprocess_exec` inherits the parent process environment by default when `env` is not specified, so setting `os.environ["TF_VAR_project_id"]` before the call is sufficient. But being explicit with `env=os.environ.copy()` is a safe pattern. Keep it simple — just set `os.environ["TF_VAR_project_id"]` before the subprocess call without changing the _run_terraform signature.
