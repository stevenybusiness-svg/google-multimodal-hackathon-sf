# Phase 1.5 — UI Design Contract

> Visual and interaction contract for the live architecture visualization. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (vanilla JS + Canvas) |
| Preset | not applicable |
| Component library | none (native Canvas API + Tailwind CDN) |
| Icon library | Material Symbols Outlined (already loaded via Google Fonts CDN) |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap inside nodes, particle radius |
| sm | 8px | Node stat text offset below label |
| md | 16px | Canvas internal padding, action card padding, node label offset below circle |
| lg | 24px | Node base radius (scaled 1.4x for Gemini = 34px) |
| xl | 32px | Gap between split-screen panels |
| 2xl | 48px | Header height (existing: 48px / h-12) |
| 3xl | 64px | Not used in this phase |

Exceptions: Node radius for Gemini Flash is 34px (24 * 1.4) to visually communicate it as the central/largest node. Touch targets not applicable -- this is a view-only visualization with no interactive nodes.

## Typography

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Node Label | 10px | 600 (semibold) | 1.2 | Inter | Node name below each circle (e.g. "Gemini", "STT") |
| Node Stat | 10px | 400 (regular) | 1.2 | Roboto Mono | Live counters below node label (e.g. "12 words", "0.85") |
| Panel Header | 10px | 600 (semibold) | 1.0 | Inter | "LIVE ACTIONS" header text, uppercase tracking-widest |
| Action Card Text | 13px | 400 (regular) | 1.4 | Inter | Action card body text in right panel |

Note: Node Label and Node Stat share the same 10px size but are differentiated by weight (600 semibold vs 400 regular) and font family (Inter vs Roboto Mono). All Canvas text uses `ctx.font` strings. Panel header and action card text use Tailwind classes matching existing UI patterns (text-[10px] font-semibold uppercase tracking-widest for headers, text-sm for card body).

## Color

All colors sourced from existing Tailwind config in `index.html`.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0d1117` (bg-dark) | Canvas background, full-screen backdrop |
| Secondary (30%) | `#161b22` (surface) | Inactive node fill, right panel background, action cards |
| Accent (10%) | See node color table below | Active node glow, edge highlights, particles |
| Destructive | `#f85149` (danger) | Not used in this phase (no destructive actions) |

### Node Color Assignments

| Node(s) | Color | Hex | Rationale |
|---------|-------|-----|-----------|
| Mic, Slack | Primary blue | `#57abff` | Audio/communication pipeline |
| STT, Dispatcher | Primary blue | `#57abff` | Core text processing path |
| Camera, Vision, Tasks | Purple | `#a855f7` | Visual/auxiliary pipeline |
| Calendar | Success green | `#3fb950` | Positive outcome / scheduling |
| Gemini Flash | Warning gold | `#e3b341` | Central decision engine, visually distinct |

### Edge Colors

| Edge Type | Inactive | Active | Style |
|-----------|----------|--------|-------|
| Primary (text path) | `#30363d` at lineWidth 2 | `#57abff` at lineWidth 3 | Solid line |
| Informing (vision/tone) | `#30363d` at 60% opacity, lineWidth 1 | `#a855f7` at 50% opacity, lineWidth 1.5 | Dashed (4px dash, 4px gap) |

### Glow Effect Specification

| Element | shadowColor | shadowBlur (idle) | shadowBlur (active peak) | Pulse frequency |
|---------|-------------|-------------------|--------------------------|-----------------|
| Active node | Node's assigned color | 0 | 20px (modulated: 12 + 8 * sin(t * 4)) | 4 Hz |
| Active edge | Edge active color | 0 | 6px | No pulse (static glow) |
| Particle | Source node color | 8px (constant) | 8px (constant) | No pulse |

### Ambient Breathing (idle state)

All nodes have a subtle 0.5 Hz breathing effect when idle: border opacity modulates by +/- 3% via `0.03 * sin(t * 0.5)`. This prevents the graph from appearing static between data events.

Accent reserved for: active node glow rings, active edge highlights, particle fills, Gemini node border when all 3 signals converge (brightest pulse).

## Canvas Rendering Specification

### Layout

| Property | Value |
|----------|-------|
| Split ratio | Left 65% (pipeline canvas) / Right 35% (action cards, w-80 = 320px) |
| Canvas sizing | Fill available width and height of left panel |
| HiDPI handling | `devicePixelRatio` scaling on canvas width/height attributes; CSS width/height at logical size; `ctx.scale(dpr, dpr)` |
| Background | Clear to `#0d1117` each frame |

### Node Rendering (front layer)

| Property | Value |
|----------|-------|
| Shape | Circle |
| Base radius | 24px (scaled by node.size, Gemini = 1.4x = 34px) |
| Inactive fill | `#161b22` |
| Active fill | Node color at 25% opacity (`+ '40'` hex suffix) |
| Inactive border | `#30363d` at 1.5px |
| Active border | Node color at 2.5px |
| Icon rendering | Material Symbols Outlined via `ctx.font = '18px Material Symbols Outlined'` centered in node circle. Fallback: text label only if font fails to render. |
| Label position | Centered, 16px below node circle bottom edge |
| Stat position | Centered, 28px below node circle bottom edge |

### Edge Rendering (back layer)

| Property | Value |
|----------|-------|
| Curve type | Quadratic Bezier (`ctx.quadraticCurveTo`) |
| Control point | Midpoint of source/target with -30px Y offset (curves upward) |
| Primary inactive | `#30363d`, lineWidth 2, solid |
| Primary active | `#57abff`, lineWidth 3, solid, shadowBlur 6 |
| Informing inactive | `#30363d60`, lineWidth 1, dashed [4,4] |
| Informing active | `#a855f780`, lineWidth 1.5, dashed [4,4], shadowBlur 6 |

### Particle System (middle layer)

| Property | Value |
|----------|-------|
| Shape | Filled circle, radius 3px |
| Color | Source node color with 8px shadowBlur glow |
| Speed | `0.008 + Math.random() * 0.004` (t parameter per frame, ~2s traversal) |
| Spawn trigger | Pipeline WS event activating an edge |
| Particles per event | 3-5 (staggered by 50ms equivalent in t-offset) |
| Removal | When t >= 1.0 (reached target node) |
| Max concurrent | No hard cap (pipeline events are sparse enough; typically < 20 active) |

### Animation Loop

| Property | Value |
|----------|-------|
| Method | `requestAnimationFrame` |
| Delta time clamping | Max 100ms (0.1s) to prevent jumps after tab switch |
| Render order | 1. Clear canvas, 2. Edges, 3. Particles, 4. Nodes |
| Target framerate | 60fps (native rAF) |
| Frame budget | < 2ms per frame |

## Interaction Contract

This phase is **view-only** -- no user interaction with the pipeline graph. The visualization responds entirely to backend pipeline events.

### Node Activation Durations

| Pipeline Event | Nodes Activated | Duration |
|----------------|-----------------|----------|
| `stt_start` | Mic | 1.0s |
| `stt_result` | STT | 1.5s |
| `understanding_start` | Gemini | 2.0s |
| `understanding_result` | Gemini | 2.0s |
| `vision_result` | Camera, Vision | 1.0s (Camera), 1.5s (Vision) |
| `action_dispatched` | Dispatcher + target (slack/calendar/tasklog) | 1.0s each |

### Edge Activation Mapping

| Pipeline Event | Edges Activated | Particles Spawned |
|----------------|-----------------|-------------------|
| `stt_start` | mic -> stt | Yes |
| `stt_result` | stt -> gemini | Yes |
| `understanding_result` | gemini -> dispatcher | Yes |
| `vision_result` | camera -> vision, vision -> gemini | Yes (both edges) |
| `action_dispatched` | dispatcher -> {action_type} | Yes |

### Live Stats per Node

| Node | Stat Label | Format | Update Source |
|------|------------|--------|---------------|
| Mic | (none) | -- | Pulses on stt_start only |
| STT | Word count | `{n} words` | Cumulative from stt_result stats.words |
| Camera | (none) | -- | Pulses on vision_result only |
| Vision | Sentiment | `{emotion} {score}` | From vision_result stats |
| Gemini | Actions found | `{n} actions` | Cumulative from understanding_result stats.actions |
| Dispatcher | Total dispatched | `{n} fired` | Increments on each action_dispatched |
| Slack | Count | `{n}` | Increments when action_type = slack |
| Calendar | Count | `{n}` | Increments when action_type = calendar |
| Tasks | Count | `{n}` | Increments when action_type = task |
