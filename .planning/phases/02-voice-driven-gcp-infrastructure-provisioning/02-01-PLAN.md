Create a new file `backend/infra.py` implementing programmatic HCL generation and async Terraform subprocess execution (per D-01, D-02, D-04, D-07, D-09).

The file must contain:

1. **Imports:**
```python
import asyncio
import logging
import os
import uuid

from backend.contracts import InfraRequest, make_action_result

logger = logging.getLogger(__name__)
```

2. **Module-level asyncio.Lock** to serialize concurrent terraform applies (per research Pitfall 3):
```python
_tf_lock = asyncio.Lock()
```

3. **TERRAFORM_DIR constant** pointing to the terraform directory:
```python
TERRAFORM_DIR = os.path.join(os.path.dirname(os.path.dirname(os.path.abspath(__file__))), "terraform")
```

4. **`generate_hcl(req: InfraRequest, project: str) -> tuple[str, str]`** function:
   - Takes an InfraRequest and GCP project ID
   - Appends a 6-char UUID hex suffix to `req["name"]` to avoid naming collisions (per research Pitfall 4): `name_slug = f"{req.get('name', 'vm')}-{uuid.uuid4().hex[:6]}"`
   - Returns a string containing ONLY resource blocks (NOT provider block — provider.tf is static):
     - `google_compute_instance` resource with: name=name_slug, machine_type (default "e2-medium"), zone (default "us-central1-a"), boot_disk with ubuntu-minimal-2210-kinetic-amd64-v20230126 image and disk_size_gb (default 20), network_interface on "default" network with access_config {}, tags=[name_slug]
     - If `req.get("ports")` is non-empty: `google_compute_firewall` resource with name=f"{name_slug}-fw", network="default", allow block for protocol="tcp" with the ports list, source_ranges=["0.0.0.0/0"], target_tags=[name_slug]
   - Use f-string with `{{` and `}}` for HCL braces
   - Returns `(hcl_content, name_slug)` tuple — hcl_content is the HCL string, name_slug is the generated resource name

5. **`_run_terraform(cmd: list[str], cwd: str) -> tuple[int, str, str]`** async function:
   - Uses `asyncio.create_subprocess_exec(*cmd, cwd=cwd, stdout=asyncio.subprocess.PIPE, stderr=asyncio.subprocess.PIPE)`
   - Awaits `proc.communicate()`
   - Returns `(proc.returncode, stdout_bytes.decode("utf-8", errors="replace"), stderr_bytes.decode("utf-8", errors="replace"))`

6. **`async def provision_infrastructure(req: InfraRequest, on_action=None) -> dict`** function:
   - Gets project from `os.environ.get("GOOGLE_CLOUD_PROJECT", "")`. If empty, return a failed action result immediately.
   - Calls `generate_hcl(req, project)` to get `(hcl_content, name_slug)`
   - Sends a "pending" action card via on_action callback if provided:
     ```python
     pending_card = make_action_result("infra", {"name": name_slug, "machine_type": req.get("machine_type", "e2-medium"), "zone": req.get("zone", "us-central1-a"), "status": "provisioning"}, "sent")
     if on_action:
         await on_action(pending_card)
     ```
   - Acquires `_tf_lock` using `async with _tf_lock:`
   - Writes hcl_content to `os.path.join(TERRAFORM_DIR, "resources.tf")` (NOT main.tf — provider.tf is static per research Pitfall 5)
   - Runs `_run_terraform(["terraform", "init", "-no-color"], TERRAFORM_DIR)` — check returncode, log and return failed action result if non-zero
   - Runs `_run_terraform(["terraform", "apply", "-auto-approve", "-no-color"], TERRAFORM_DIR)` — check returncode
   - On success: return `make_action_result("infra", {"name": name_slug, "machine_type": req.get("machine_type", "e2-medium"), "zone": req.get("zone", "us-central1-a"), "status": "provisioned"}, "sent")`
   - On failure: return `make_action_result("infra", {"name": name_slug, "status": "failed"}, "failed", error=f"terraform apply failed: {stderr[:300]}")`
   - Wrap everything in try/except, return failed action result on any exception
   - Log at INFO level for key steps (generating HCL, running init, running apply, success/failure)

CRITICAL anti-patterns to avoid:
- Do NOT use `subprocess.run()` — use `asyncio.create_subprocess_exec`
- Do NOT generate the provider block in generate_hcl() — provider.tf is static and committed
- Always use `-auto-approve` and `-no-color` flags
- Always acquire `_tf_lock` before writing resources.tf or running terraform commands
