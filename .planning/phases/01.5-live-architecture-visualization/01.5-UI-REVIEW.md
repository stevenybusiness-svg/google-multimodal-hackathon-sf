## Top 3 Priority Fixes

1. **Missing "Pipeline disconnected. Reconnecting..." error copy** — When WS drops, `stopMeeting('disconnected')` navigates away immediately; users on the pipeline screen see no in-place feedback before the screen transitions. Fix: in `session.js`, when `reason === 'disconnected'` and `state.currentScreen === 'pipeline'`, set `dom.pipelineStatusText.textContent = 'Pipeline disconnected. Reconnecting...'` and add a brief delay before navigating to give visual context.

2. **Node label vertical offset is 8px below circle edge instead of spec's 16px** — `pipeline.js:209` uses `pos.y + baseRadius + 8`; spec says 16px (`md` spacing token). The smaller gap crowds the label against the circle and compresses space for stat text. Fix: change to `pos.y + baseRadius + 16`.

3. **Stat text vertical offset is 20px instead of spec's 28px** — `pipeline.js:216` uses `pos.y + baseRadius + 20`; spec says 28px (stat 28px below circle bottom edge). This causes stat text to overlap the label at larger font scales. Fix: change to `pos.y + baseRadius + 28`.

### Pillar 4: Typography (3/4)

**Canvas fonts — exact spec match:**
| Role | Spec | Impl |
|------|------|------|
| Node Label | `600 10px Inter` | `'600 10px Inter, sans-serif'` (`pipeline.js:206`) |
| Node Stat | `400 10px Roboto Mono` | `'400 10px Roboto Mono, monospace'` (`pipeline.js:214`) |
| Icon | `18px Material Symbols Outlined` | `'18px Material Symbols Outlined'` (`pipeline.js:201`) |

**HTML panel typography — matching spec:**
- Panel header: `text-[10px] font-semibold uppercase tracking-widest` — matches spec's "10px, 600, uppercase, tracking-widest" (`index.html:291`)
- Action card body: `text-xs text-slate-300` (13px on Tailwind CDN defaults = 12px; `text-sm` would be 14px) — minor: spec calls for `13px` Action Card Text but `text-xs` is 12px. The gap is minimal and aligns with existing meeting screen patterns.

**Consistency issue:** Pipeline panel header uses `font-semibold` while all other panel section headers in the file use `font-bold` (`Live Transcript:228`, `Detected Actions:240`, `Transcript Log:320`). The spec calls for `600 semibold` — so the pipeline panel is technically correct but creates visual inconsistency with the rest of the app. This is a pre-existing app pattern issue, not a pipeline-introduced regression.

**Font sizes in use (whole app):** `text-[10px]`, `text-xs`, `text-sm`, `text-base`, `text-lg`, `text-xl`, `text-4xl` — 7 distinct sizes. Higher than ideal (>4) but driven by the multi-screen nature of the full app; the pipeline screen itself uses only 3 distinct sizes (`text-[10px]`, `text-sm`, `text-xs`).

### Pillar 5: Spacing (3/4)

**Matching spec:**
- Right panel width: `w-80` (320px) — matches spec (`index.html:289`)
- Header height: `h-12` (48px) — matches spec (`index.html:254`)
- Canvas container padding: `p-4` (16px) — matches spec's md=16px (`index.html:280`)
- Action panel padding: `px-4 py-2` — matches spec pattern
- Mic level bar: `h-[6px]` — matches spec (`index.html:273`)
- Pipeline canvas container: `flex-1` + `min-h-0` — correct flex sizing
- Node base radius: `24px` — matches spec (`pipeline.js:161`)
- Gemini radius: `24 * 1.4 = 33.6px` (spec: 34px) — 0.4px rounding, negligible
- Particle radius: `3px` — matches spec

**Deviations:**
- **Label offset:** `pos.y + baseRadius + 8` — spec declares 16px (md token) below circle bottom edge. Implementation places the label 8px below, half the spec value. (`pipeline.js:209`)
- **Stat offset:** `pos.y + baseRadius + 20` — spec declares 28px below circle bottom edge. Implementation places stats at 20px, creating compressed vertical stacking. (`pipeline.js:216`)

Both offsets are likely a pragmatic adjustment (tight canvas space with many nodes), but they deviate from the declared spacing scale.

**Arbitrary values in HTML (pipeline section):** `[10px]`, `[12px]`, `[6px]` — all justified: 10px for micro-labels, 12px for small icons, 6px for the mic bar. These appear elsewhere in the app and are consistent with the established UI pattern.

### Pillar 6: Experience Design (4/4)

**State coverage — excellent:**
- **Empty state:** Pipeline canvas shows "Waiting for pipeline activity..." overlay with correct pointer-events-none; hidden on first pipeline event (`pipeline.js:330-332`, `index.html:283-286`)
- **Loading state:** Status chip shows "Connecting..." on load; updates to "Listening..." via `handleStatusMessage` mirrored to `pipelineStatusText` (`render.js:474`)
- **Animation lifecycle:** `startAnimation()` called on meeting start, `stopAnimation()` on meeting stop (`session.js:83`, `93`)
- **Font readiness:** `document.fonts.ready.then(startAnimation)` ensures ambient breathing before any pipeline events (`pipeline.js:393`)
- **HiDPI:** Full `devicePixelRatio` support with `ctx.setTransform` reset on resize prevents blur on retina displays (`pipeline.js:52-64`)
- **Responsive canvas:** `ResizeObserver` on parent element, trigger on screen switch via `window.dispatchEvent(new Event('resize'))` (`render.js:25`)
- **Frame-rate independence:** `dt * 60` multiplier for particle speed; delta-time clamped to 100ms max (`pipeline.js:245`, `279`)
- **All 6 backend event types handled:** `stt_start`, `stt_result`, `understanding_start`, `understanding_result`, `vision_result`, `action_dispatched` all mapped in `EVENT_MAP` (`pipeline.js:315-322`)
- **Dynamic edge routing for `action_dispatched`:** Correctly routes to `slack`, `calendar`, or `tasklog` based on `action_type`; handles `document` type by routing to `slack` (`pipeline.js:354`)
- **Live stats:** All 7 stat-bearing nodes update correctly (`pipeline.js:372-386`)
- **Action card mirroring:** Cards appear in both meeting screen and pipeline panel (`session.js:30-33`)
- **Mic level mirroring:** `pipelineMicLevel` width updated alongside `micLevel` in audio processor loop (`media.js:194-196`)
- **Stop button:** `pipeline-stop-btn` wired to `stopMeeting('user')` (`session.js:141-143`)

**Only gap (covered in Pillar 1):** No in-place WS disconnection visual on pipeline screen — routes immediately to meeting end flow without showing "Pipeline disconnected. Reconnecting..." in the status chip.
