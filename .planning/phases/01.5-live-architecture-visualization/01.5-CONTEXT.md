# Phase 1.5: Live Architecture Visualization - Context

**Gathered:** 2026-03-23
**Status:** Ready for planning

<domain>
## Phase Boundary

Add a live pipeline visualization to the existing meeting screen. The Canvas-based pipeline graph supplements (not replaces) the current meeting view. Pipeline takes the left panel with actions on the right. Transcript view remains but is made smaller. Auto-shows when meeting starts — no toggle needed.

</domain>

<decisions>
## Implementation Decisions

### Layout
- Pipeline view SUPPLEMENTS the meeting screen (not replaces)
- Pipeline canvas takes the left panel
- Action cards stay on the right panel
- Transcript view is made smaller (collapsed or reduced height below the canvas, or overlaid)
- Pipeline auto-shows when meeting starts — no toggle or nav tab needed

### Node Icons
- Use Material Symbols Outlined font for node icons in Canvas (`ctx.font = '18px Material Symbols Outlined'`)
- Font is already loaded via Google Fonts CDN in the project

### Vision Pipeline Events
- Forward vision results to WebSocket for pipeline animation (recommended by research)
- Emit `pipeline` event with `vision_result` subtype when `analyze_frame` completes
- Reuse existing `/ws/audio` WebSocket connection — no new endpoint

### Claude's Discretion
- Exact transcript sizing/positioning relative to pipeline canvas
- How to integrate pipeline canvas into existing meeting screen DOM structure
- Animation timing tuning within the ranges specified in UI-SPEC
- Node position coordinates (research provides defaults; adjust for actual canvas size)

</decisions>

<specifics>
## Specific Ideas

- Pipeline graph has ~10 nodes in fixed positions — no dynamic layout algorithm needed
- Particle system: 3-5 particles per edge activation, ~2s traversal time
- Gemini node is 1.4x larger than others (central decision engine)
- Primary edges (text path) are solid and thick; informing edges (vision/tone) are dashed and thin
- All nodes have ambient 0.5 Hz breathing effect when idle
- Action cards in right panel should mirror existing detected actions format

</specifics>
